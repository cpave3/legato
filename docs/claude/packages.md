## Key Packages

- `internal/engine/store/` — SQLite store with embedded migrations, task/column mapping/sync log CRUD, task ID generation, state interval tracking (`RecordStateTransition`, `GetStateDurations`, `GetStateDurationsBatch`, `GetStateTimeline` for sparkline bucket sequences). `DeleteTask` also clears `state_intervals` for the task and any swarm sub-task children (the FK cascade was removed in migration 021)
- `internal/engine/macros/` — Tiny shared package defining the `Macro` struct (`Name`, `Keys`) and `ListResult` JSON envelope used by the `GET /api/macros` endpoint, the TUI macro picker overlay, and the YAML config layer
- `internal/engine/events/` — Channel-based event bus with buffered pub/sub (buffer size 64, non-blocking drops on full), error event types (`EventSyncError`, `EventTransitionFailed`, `EventAuthFailed`, `EventRateLimited`) with `ErrorPayload` struct
- `internal/engine/auth/` — Web UI auth token management. `EnsureToken(dataDir)` reads `<dataDir>/auth-token` or generates 32-byte `crypto/rand` hex token on first run. `RegenerateToken(dataDir)` overwrites with new token. `ReadToken(dataDir)` reads without generating. Token file is 0600 permissions
- `internal/engine/certs/` — Auto-generated TLS certificates. `EnsureCerts(dataDir, extraDNS...)` creates a self-signed ECDSA CA + server cert in `<dataDir>/certs/`, reuses existing if valid (not expiring within 30 days, all required DNS SANs present). Server cert SANs include localhost, loopback, all local network IPs via `net.InterfaceAddrs()`, and any extra DNS names passed in. `loadOrCreateCA` reuses existing CA across server cert regenerations. All stdlib crypto — no external dependencies
- `internal/engine/jira/` — Jira REST API v3 client (types, HTTP client with Basic Auth/backoff, ADF-to-Markdown converter), plus `Provider` adapter
- `internal/engine/github/` — GitHub PR status client via `gh` CLI: `FetchPRStatus` (single branch), `BatchFetchPRStatus` (concurrent fan-out with semaphore(5)), `DetectRepo`/`DetectBranch` (git remote/branch detection), `FetchCommentCount` (via `gh api`), `deriveCheckStatus` (CI rollup aggregation). LookPath + ExecCommand injectable for testing.
- `internal/engine/tmux/` — Tmux session manager: spawn, kill, capture-pane, attach, set-option, list/filter legato-prefixed sessions, `PaneCommands()` (batch query live foreground process names via `tmux list-panes -a -f`). LookPath-injectable for testing.
- `internal/service/` — BoardService (columns/cards CRUD + CreateTask + DeleteTask + UpdateTaskDescription + UpdateTaskTitle + UpdateTaskWorkspace + ListCardsByWorkspace + ListWorkspaces + ArchiveDoneCards + ArchiveTask + CountDoneCards), SyncService (pull/push with conflict resolution, provider→task conversion, SearchRemote + ImportRemoteTask), AgentService (tmux session lifecycle + SQLite tracking + dynamic pane command population + task title enrichment + state duration queries via `GetTaskDurations` + per-task sparkline data via `GetStateTimeline`), PRTrackingService (branch-to-task linking, periodic PR status polling via `gh` CLI, auto-link on agent spawn), ReportService (GenerateReport → *Report aggregating ByDay/ByTask/ByWorkspace/BySwarm/ByDirectory time spent), `TicketProvider` interface for pluggable backends, `SeedWorkspaces` (config→DB workspace seeding)
- `internal/setup/` — Setup wizard logic: first-run column seeding, interactive Jira configuration (credential validation, project/status discovery, column mapping heuristics, config writing), `ColumnSeeder` interface for testability
- `internal/tui/` — Root Bubbletea app model, view routing (viewBoard/viewDetail/viewAgents), overlay state management (`overlayType` enum + `activeOverlay tea.Model`), EventBus bridge
- `internal/tui/agents/` — Agent split-view: sidebar with agent list (status, task ID, title, command per entry) + terminal output panel, j/k navigation, spawn/kill/attach keybindings, capture-pane polling at 200ms
- `internal/tui/board/` — Kanban board model with vim navigation, card/column rendering, agent status line with duration display
- `internal/tui/detail/` — Full-screen task detail view with Glamour markdown, metadata header (provider-specific fields from RemoteMeta), viewport scrolling, `e` to edit description via `$EDITOR` (local tasks), `t` to edit title via overlay (local tasks)
- `internal/tui/clipboard/` — OS-native clipboard (pbcopy/wl-copy/xclip/xsel) and browser open (open/xdg-open)
- `internal/tui/overlay/` — Overlay system: shared `RenderPanel`, move overlay (single-letter shortcuts, `w` transitions to workspace assignment), search overlay (real-time filtering via `BoardService.SearchCards`), create overlay (inline task creation with title/description/column/priority/workspace), delete overlay (confirmation with remote/local distinction), import overlay (remote ticket search + import via `SyncService.SearchRemote`), title edit overlay (pre-filled text input for local tasks), workspace overlay (board view filter), move-workspace overlay (task workspace assignment), archive overlay (bulk archive done cards with count + confirmation), link PR overlay (two-phase: repo+number input → preview → confirm), help overlay (keybinding reference), agent action overlay (`Shift+M` from the agents view; role-aware Send message / Close worker / Finish swarm; routes through `SwarmService.Message`/`MessageParent`/`Close`/`Finish` via the app handlers; swarm-only — no-op for solo agents), macro picker overlay (`m` from the agents view; lists `macros:` config entries and sends the chosen `Keys` payload to the focused agent's tmux pane via `SendKeysLine`/`SendKeysMultiline`)
- `internal/tui/statusbar/` — Status bar with sync state, relative time, key hints, warnings, error messages (auto-clear on sync success)
- `internal/tui/theme/` — Lipgloss color palette, style constants, and icon system (`icons.go`: `NewIcons("unicode"|"nerdfonts")` for provider/agent/warning glyphs)
- `internal/engine/ipc/` — Unix domain socket IPC: `Server` (listens, parses newline-delimited JSON, calls callback), `Send` (client, silent on missing socket), `Broadcast` (sends to all `*.sock` in `SocketDir()`), `SocketPath()` (PID-based, XDG_RUNTIME_DIR with /tmp fallback), `Message` struct (type/task_id/status/content)
- `internal/engine/hooks/` — AI tool hook script generation + swarm adapter capabilities.
  - `ClaudeCodeAdapter` implements `service.AIToolAdapter`: generates `.claude/hooks/` scripts, merges/removes `.claude/settings.json` entries. Also implements `service.LaunchCommandAdapter` (`claude --append-system-prompt "$(cat $LEGATO_ROLE_PROMPT_FILE)"` plus user `launch_args`) and `service.RolePromptingAdapter`. Does not implement `LaunchSelfKickoff` — claude needs the kickoff send-keys because `--append-system-prompt` is system context, not a user turn.
  - `ChimeraAdapter` similarly implements `AIToolAdapter`, `LaunchCommandAdapter` (`chimera --prompt "$(cat $LEGATO_ROLE_PROMPT_FILE)" [--mode <name>] [user launch_args]`), `RolePromptingAdapter`, and `RolePromptPreambleAdapter` (sandbox-mode warning prepended to every role prompt). Does implement `LaunchSelfKickoff` returning true — chimera's `--prompt` makes the role prompt content turn 1, so the agent self-starts and the kickoff send-keys is skipped. `SetModes(map[string]string)` opts into per-role `--mode <name>` injection from `cfg.Adapters.chimera.modes`; legato ships no default mode files (users opt in).
  - `StaccatoAdapter` generates `post-pr-create` hook at `~/.config/staccato/hooks/post-pr-create/legato-pr-link.sh` (directory-based, no config file).
  - Embedded role prompts under `prompts/conductor.md` and `prompts/worker.md`; `resolveRolePrompt(overrides, role)` returns the conductor prompt for `"conductor"` and the worker prompt for any non-empty role label, honoring `SetRoleOverrides(RolePromptOverrides)` first.
  - `SetLaunchArgs([]string)` on `ClaudeCodeAdapter` and `ChimeraAdapter` appends user-supplied flags to the launch command (shell-quoted via `shellQuote`).
  - `SetTiers(map[string][]string)` on `ClaudeCodeAdapter` and `ChimeraAdapter` registers per-tier launch_args. `LaunchCommand(env, brief, tier)` appends the matching tier's args after the base launch_args (so tier-specified flags like `--model` win on conflicts); a `tier != ""` lookup that misses logs a warning and falls back to base args.
- `internal/engine/swarm/` — File-ownership scope detection for swarm orchestration. `MatchScope(globs, path)` and `ScopeOverlaps(a, b, root)` walk the working tree and detect file-level overlaps using `github.com/bmatcuk/doublestar/v4`. Skips `.git`, `node_modules`, `vendor`, `dist`, `build`, `.next`, `.svelte-kit`, `.turbo`, `.cache`. `ValidateGlob`/`ValidateScope` reject malformed patterns
- `internal/service/swarm.go` — `SwarmService` is the conductor-driven orchestration surface. Conductor methods: `StartSwarm`, `ApplyApprovedPlan`, `Dispatch`, `Message`, `MessageParent`, `Broadcast`, `Close`, `Finish`. Worker methods: `Progress`, `Question`, `Built`. Pending-plan persistence: `InsertPendingPlan`/`GetPendingPlan`/`ListAllPendingPlans`/`DeletePendingPlan` back the swarm-pending-plans table that survives server restarts and PWA tab suspends. `HandleAgentDied` + `StartEventLoop` subscribe to `EventAgentDied` and transition workers to `cancelled` while notifying the conductor. Per-worker progress debouncer (1s window) collapses chatty workers; `built`/`question`/`died` bypass debounce. `Snapshot(parentID)` returns JSON coordination surface; `ListSubtaskInfos` returns UI-friendly views. `LatestSnapshot(parentID)` returns a sub-millisecond in-memory `*SwarmSnapshot` (Total/Done/Cancelled/Active counters); mutated incrementally by `bumpSnapshot` on every status change path (`Dispatch`, `Built`, `Close`, `Finish`, `HandleAgentDied`, `Progress`, `Question`). On a cold cache (e.g. after a server restart), the first read lazily rebuilds from `ListSubtasksByParent` so Total/Done are correct. `ApplyApprovedPlan` re-runs `swarm.ValidatePlan` against `SwarmConfig.ValidateOptions` before persisting so the TUI/web edit-and-approve path can't bypass tier/scope checks. `internal/service/tier_catalog.go::formatTierCatalog(map[string]map[string]string)` renders the "Available tiers" markdown section that `StartSwarm` appends to the conductor brief. `AgentDiedPublisher` is a thin adapter so `agentService` can publish `EventAgentDied` without importing the events package directly. Send-keys delivery to swarm participants goes through the agent service's `Tmux()` accessor — `SendKeysLine` for short messages, `SendKeysMultiline` for multi-line payloads (auto-base64-wrapped). See `docs/claude/swarm.md` for the full lifecycle
- `internal/cli/` — CLI subcommand handlers: `TaskUpdate` (move task to column, case-insensitive status), `TaskNote` (append timestamped note), `AgentState` (update agent activity), `AgentSummary` (tmux-formatted agent activity counts with `--exclude` flag), `AgentStatus` (swarm-aware tmux status-line markup for a single task — falls back to `AgentSummary` for solo agents; reads `state_intervals` + `swarm_subtasks` + `swarm_events` on every call since the in-process `SwarmService.LatestSnapshot` cache is unreachable from a separate CLI process), `TaskLink` (link branch/repo to task for PR tracking), `TaskUnlink` (remove PR association). All broadcast IPC to all running instances. Used by `cmd/legato/` subcommand dispatch.
- `internal/server/` — HTTP + WebSocket server for web UI. `GET /health` (board state JSON), `GET /api/agents` (includes a per-agent `state_timeline` field — bucket sequence over `SetSparklineWindow` window, default 10m / 10 buckets), `GET /api/tasks` (legacy column-grouped board), `GET /api/macros` (returns the configured `macros:` list as `{"macros": [{name, keys}, ...]}`; auth-protected), `POST /api/agents/spawn` (ephemeral agent creation), `POST /api/agents/kill` (agent termination), `/ws` (WebSocket). `SetMacros([]macros.Macro)` and `SetSparklineWindow(window, buckets)` configure the per-instance state on the `*Server` struct before `Start()`/`Serve()`. Auth middleware checks `Authorization: Bearer <token>` on API routes and `?token=` query param on WebSocket; exempt: `GET /health`, `OPTIONS`, static assets. CORS middleware sets `Access-Control-Allow-Origin: *` on all responses. `streamManager` fans out `pipe-pane` output per agent with deferred pipe start after first resize. After pipe starts, bounces pane width by 1 column to force SIGWINCH redraw through pipe (prevents snapshot/stream divergence). Output chunks sent synchronously to preserve ordering. Prompt detection via `prompt.Detect` (500ms debounce after output settles + on-demand `detect_prompt` message). `resizePane` computes min of web client sizes + attached tmux terminal clients. `send_keys` supports literal text (`\n`-terminated) and space-separated named key sequences (e.g. `"Down Enter"`). `refresh_pane` recaptures pane with ANSI escapes, restores cursor position via tmux `display-message` cursor query, and resets web xterm. Escape sequence buffering (`findIncompleteEscape`) prevents split ANSI sequences from corrupting xterm.js state. `Server.Serve(ln)` accepts pre-bound listener for auto-start use. `onStreamEnd` callback triggers `ReconcileSessions` + `agents_changed` broadcast when pipe-pane EOF detected (shell exit)

**Board endpoints.** `GET /api/board` returns the full kanban board (columns, cards, workspaces, PR metadata, swarm stats, agent state). `GET /api/workspaces` returns workspace configuration. `POST /api/tasks` creates a local task. `GET /api/tasks/{id}` returns card detail (with optional `?format=full` for plain-text context export). `PATCH /api/tasks/{id}` edits title, description, workspace, or column. `DELETE /api/tasks/{id}` removes the task. `POST /api/tasks/{id}/archive` archives a single done task. `POST /api/board/archive-done` bulk-archives all done cards. `GET /api/tasks/search?q=query` searches cards by title (min 2 chars). `POST /api/tasks/{id}/link-pr` links a task to a GitHub PR. `POST /api/tasks/{id}/unlink-pr` removes the link. `GET /api/tasks/{id}/pr-preview` fetches live PR metadata by owner/repo/number. `GET /api/repo/detect` detects the current repo from git context. `GET /api/remote/search?` searches remote tickets via SyncService (min 2 chars). `POST /api/remote/import` imports a remote ticket. Board event broadcasting via `StartBoardEvents()` subscribes to `EventCardsRefreshed`, `EventCardMoved`, and `EventCardUpdated` and broadcasts a `cards_changed` WebSocket message to all connected clients. `SetSyncService`/`SetPRTrackingService` wire optional sync and PR-tracking services respectively.
- `internal/server/swarm.go` — HTTP handlers for swarm verbs: `POST /api/swarm/start` (spawn conductor), `POST /api/swarm/dispatch`, `POST /api/swarm/message`, `POST /api/swarm/broadcast`, `POST /api/swarm/close`, `POST /api/swarm/finish`, `GET /api/swarm/status/<parent-id>`, `GET /api/swarm/inbox/<parent-id>`, `GET /api/swarm/pending-plan/<parent-id>`. All wrap `SwarmService` methods and return structured JSON (`{"status":"ok"}` or the swarm snapshot). Error responses follow the same `{"error":"<message>"}` shape as other API endpoints.
- `internal/server/ws.go` — WebSocket message types extended with swarm events: `plan_proposed` (server→client, carries plan summary + reply socket path), `plan_verdict` (client→server, carries parent task ID + status + notes + plan path), `swarm_changed` (server→client, triggers refetch of swarm status in the UI). The server subscribes to IPC `EventPlanProposed` and `EventSwarmChanged` on the event bus at startup and fans them out to all connected WebSocket clients via `hub.Broadcast`.
- `config/` — YAML config parser with env var expansion (`os.ExpandEnv`) and XDG path resolution
